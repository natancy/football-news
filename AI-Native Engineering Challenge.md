# AI-Native Development Challenge

---

# Objective

Build a small but complete software project using AI-native development practices and the AI tooling of your choice.

The goal is not to build the most sophisticated game.

The goal is to experience and document the full AI-native software development lifecycle:

* Idea and requirements

* Planning and specification

* Architecture and design

* Implementation

* Testing and validation

* Documentation

* Iteration and refinement

---

# The Challenge

Build a playable card game or another simple game of your choice.

Examples include:

* Durak (Fool)

* Blackjack

* Poker variant

* Hearts

* Uno-style game

* Tic-Tac-Toe

* Connect Four

* Chess mini-engine

* Word game

* Browser game

* Any similar project

Choose something small enough to complete within the challenge timeframe but large enough to require planning, implementation, testing, and iteration.

# Evaluation

This is a learning exercise, not a coding competition.

Projects will be evaluated primarily on:

* Completeness

* Effective use of AI

* Quality of AI-native workflow

* Documentation quality

* Lessons learned

* Reusability of insights for the broader organization

We are interested in understanding not only what worked, but also what did not work.

# Optional Bonus

Create a short 3–5 minute demo video showing:

* The finished game

* Repository structure

* AI workflow used

* Key learnings

**Or even better (especially if you are an Engineer\!):**

Please consider making your game playable directly from GitLab/GitDocs, not only locally.

The goal is simple: someone should be able to open your README.md, click a link, and play the game without cloning the repo, installing dependencies, or running a local server.

This is not required for the MVP, but it is a useful extra step because it makes demos, reviews, and sharing much easier. It also exercises a more complete AI-native workflow: build, CI/CD, deployment, troubleshooting, and documentation.

**Suggested ask for your AI tool:**

*Add GitLab/GitDocs deployment for this game and update the README with a playable link. The game should be playable from GitLab without requiring a local clone/build/run.*

Depending on your project and GitLab setup, you may need to configure GitLab Pages/GitDocs, CI runner settings, or a token/credential. Please do not share tokens in the repo or in prompts. Store credentials only in the appropriate local or GitLab CI/CD variable settings.

Again, this is a bonus, not a blocker. But it is a great way to turn the project from “works on my machine” into something others can actually try with one click.

# Expected Outcome

By the end of the challenge, we should have:

* Practical experience across multiple AI tools and workflows

* Examples of successful AI-native development patterns

* Lessons learned from real projects

* Input for future AI-native product, dev and design playbooks and standards

Most importantly, we want to better understand how engineers can effectively partner with AI to deliver software end-to-end and help RingCentral continue its transition toward becoming an AI-native P\&T organization.
